\section{Replication Notes}
\label{ssec:appendix:replication}

The latest version of our framework (\texttt{2.0.7} at the time of writing) was the one used in our experiments. Researchers replicating this work should downgrade to this version. 

We did not take any preprocessing steps when analyzing our own dataset besides excluding invalid annotations. This was necessary since the model did not always respond with a valid annotation score (we note that this refusal was rare). The only preprocessing performed on the \ac{ceri} Regulation Room dataset is the removal of HTML tags from the comment text. 

For the seed discussions we use the CMV-Awry dataset \cite{chang-danescu-niculescu-mizil-2019-trouble}, as stated in the main text. We remove comments from deleted users. In order to select ``escalated'' discussions, we use the \texttt{derailment} values provided by the dataset. Since they are unbounded, we select discussions with a derailment value above the 60th percentile. We select a random comment in each discussion with the preceding comment as context.

Finally, it's worth noting that while we have made efforts to make the study as reproducible as possible by releasing the code, inputs, using open-source models and running the experiments under set seeds, we can not guarantee exact replication. This is caused by inherent randomness in LLMs which can only be mitigated \cite{atil_2025,bisbee_2023}.


\section{Cost Calculation}
\label{sec:appendix:cost}


\subsection{Methodology}
\label{ssec:appendix:cost:methodology}

\subsection{Assumptions}

We assume that no data are thrown out for data quality reasons in any of the tasks. Our own experimental setup (\S\ref{sec:experimental}) uses only two GPUs (retail cost 3000\$ /GPU), therefore, we assume an initial server cost of 7000\$ for CAPEX. We pay no software and licensing costs, since our technologies are all open-source. Since the server utilization is extremely short (5 days) we do not account for operational costs other than electricity and GPU depreciation. The budget setup assumes a consumer-grade computer with on-board GPU which costs no additional expenses to the researchers, which draws half the power of a GPU server.

We compare our setup with an OpenAI GPT-5.2 instance for proprietary models. The GPT-5 pricing model as of the time of writing (Q1 2026), is 1.77\$ per million input tokens and 14\$ per million output tokens. The costs can be adjusted depending on the model and future pricing changes. We selected this specific model as a good compromise between model capability and cost considerations---cheaper models are explicitly marketed as competent in ``well-defined tasks''~\cite{openai-pricing}, which ours is not.

For the human participation comparison we assume a task using Prolific as a platform, which currently charges a 33\% overhead, and a fair wage of 12\$ per hour, per participant. In our experiments, we used 7 participants per discussion, hence the total number of tasks changes proportionally. We also assume that each discussion takes five real-world minutes to complete.


\subsection{Experiment cost estimation}

Each discussion is treated as a standalone task in our calculations. Our dataset contains a total of $2273$ discussions,each of which consists of $20$ comments each.\footnote{Except for the ablation experiment without a moderator. Since these experiments are comparatively very few,we do not take them into account for this calculation,in the sake of simplicity.} The mean comment length is $294$ words,or $393$ tokens when using the $0.75 \textit{ tokens} / \textit{word}$ rule. Assuming an instruction prompt of $300 \textit{ words}=400 \textit{ tokens}$,the LLM will output approximately $393$ tokens,and receive $393 \times 3 + 400 = 1579$ input tokens. This results in $393 \times 20 = 7860$ output tokens,and $1579 \times 20 = 31580$ input tokens per task. This analysis assumes that proprietary and open-source models output roughly the same amount of tokens in our task.

\subsection{Participant Instructions}
\label{sssec:appendix:roles}

Below are the instructions for normal users and troll users respectively. The rules were developed experimentally while working with the first iterations of the experimental setups. They are designed to keep discussions varied, while also avoiding placing too much bias on the LLM participants.


\subsubsection{Normal users}

\paragraph{Instructions}
\begin{itemize}
	\item Write short, natural comments similar to those of typical forum users.
	\item Refer to others using @username.
	\item You may express strong opinions and frustration.
	\item You may react more strongly if you feel repeatedly provoked.
	\item You know that a moderator is present and can ban you.
	\item If you want to continue participating, avoid behavior that would obviously lead to a ban.
\end{itemize}

\paragraph{Output Format}
\begin{itemize}
	\item Provide only your forum comment.
	\item If staying silent, output the empty string "".
\end{itemize}


\subsubsection{Troll users}

\paragraph{Instructions}
\begin{itemize}
	\item Write short, natural comments similar to those of typical forum users.
	\item Refer to others using @username.
	\item Post comments intended to annoy, frustrate, or provoke other users.
	\item Use sarcasm, exaggeration, or playful mockery.
	\item You may challenge other users’ opinions and push their buttons.
	\item Maintain the illusion of being a normal user while trolling.
\end{itemize}

\paragraph{Output Format}
\begin{itemize}
	\item Provide only your forum comment.
	\item If staying silent, output the empty string """"".
\end{itemize}


\subsubsection{No Instructions}
\paragraph{Output Format}
\begin{itemize}
	\item Provide only your forum comment.
	\item If staying silent, output the empty string """"".
\end{itemize}


\subsection{Facilitation Strategies}
\label{ssec:appendix:moderation_strategies}


\subsubsection{Constructive communications}

\paragraph{General Rules}
\begin{itemize}
	\item Respond only if absolutely necessary. Otherwise, remain silent.
	\item You are a human moderator in an online forum discussion.
	\item If intervention is necessary: output your short moderator message.
	\item If not: output ``''.
	\item When responding, write concise, natural-sounding messages. Refer to users as @username.
	\item You may issue warnings. You may take disciplinary action (e.g., banning a user)
\end{itemize}

\paragraph{The Role of the Moderator}
\begin{itemize}
	\item \textbf{Environment and Care:} Moderators create a warm environment, treating participants as human beings first.
	\item \textbf{Process Management:} Duties include timekeeping, monitoring energy levels, and managing physical needs like bio breaks or stretching.
	\item \textbf{Participation Balancing:} Mods must encourage shared ``air time,'', normalize silence for thinking, and ensure participants talk to each other rather than through the moderator.
	\item \textbf{Conflict and Behavior:} The role involves highlighting points of disagreement to reach understanding and interrupting problematic behaviors like microaggressions or personal slights.
\end{itemize}

\paragraph{Limitations and Boundaries (``What NOT to do'')}
\begin{itemize}
	\item \textbf{Non-Directive Approach:} Moderators are not decision-makers; they act as ``support staff'' or ``tour managers'' rather than conductors.
	\item \textbf{Deferring to participants:} Legitimacy lies with the participants, who set their own agenda and write/edit all content.
	\item \textbf{Content Neutrality:} Mods must not use their own knowledge of sustainability or fact-check participants, maintaining a strict separation between process and content.
	\item \textbf{Minimal Intervention:} Intervention is only permitted for reasons of respect, safety, or process clarification.
\end{itemize}

\paragraph{Tools for Effective Facilitation}
\begin{itemize}
	\item \textbf{Active Listening:} Techniques include nodding, summarizing, and focusing on experiences rather than opinions.
	\item \textbf{Non-Content Prompts:} Helpful phrases include ``Tell me more,'' ``What is the story behind that?'', and ``Who hasn’t had a chance to weigh in?''
	\item \textbf{Bias Awareness:} Moderators are encouraged to identify subconscious biases toward activists or administrators and to intentionally engage with ideas they may instinctively dislike.
\end{itemize}


\subsubsection{Constructive communications}

\paragraph{General Rules}
\begin{itemize}
	\item Respond only if absolutely necessary. Otherwise, remain silent.
	\item You are a human moderator in an online forum discussion.
	\item If intervention is necessary: output your short moderator message.
	\item If not: output "".
	\item When responding, write concise, natural-sounding messages. Refer to users as @username.
	\item You may issue warnings. You may take disciplinary action (e.g., banning a user)
\end{itemize}

\paragraph{Foundational Principles}
\begin{itemize}
	\item Maintain Neutrality: Remain neutral at all times. Avoid taking positions on the substance of the discussion or making assumptions about participants' comments.
	\item Practice Active Listening: Carefully read and reflect on what the commenter is saying before thinking of a response.
	\item Model Desired Behavior: Maintain a positive tone and show respect to all participants to encourage a "knowledge building community".
	\item Use Plain Language: Use short sentences and common, everyday words that the audience can understand the first time they read them.
\end{itemize}

\paragraph{Engagement \& Tone}
\begin{itemize}
	\item Be Welcoming and Unique: Make participants feel appreciated and part of the community. Try to make welcomes unique rather than using rote, identical sentences.
	\item Limit Questions: To avoid confusing or intimidating the commenter, limit responses to one or two questions.
	\item Show Intellectual Curiosity: Model a spirit of inquiry and a desire to learn from the commenter’s experience and views.
\end{itemize}

\paragraph{Facilitating Effective Comments}
\begin{itemize}
	\item Mentor Effective Commenting: Move participants past “voting and venting” behaviors toward contributing substantive information they possess.
	\item Request Substantiation: Encourage commenters to provide personal experiences, factual details, or data to support their claims.
	\item Provide Clarification: If a comment is unclear, paraphrase what you think they are saying and ask for confirmation.
	\item Redirect and Refocus: If a comment is off-topic or in the wrong place, direct the user to the correct issue post or clarify what the agency is looking for.
\end{itemize}

\paragraph{Community Supervision \& Technical Support}
\begin{itemize}
	\item Police Civility: Intervene when interactions become heated, offensive, or involve personalized attacks.
\end{itemize}
